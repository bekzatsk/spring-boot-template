    **RefreshTokenService.kt** — Create new file at `src/main/kotlin/kz/innlab/template/authentication/RefreshTokenService.kt`. Follow research Pattern 7 closely. The service has three public methods:

    1. `createToken(user: User): String` — Generate 32 random bytes via `SecureRandom`, Base64url-encode (no padding) to get the raw token. SHA-256 hash the raw token, Base64-encode the hash. Save a new `RefreshToken` entity with `tokenHash=hash`, `expiresAt=Instant.now().plus(expiryDays, ChronoUnit.DAYS)`, `user=user`. Return the raw token (sent to client). Mark `@Transactional`.

    2. `rotate(rawToken: String): Pair<User, String>` — Hash the incoming token. Look up by hash (`findByTokenHash`). If not found, throw `BadCredentialsException("Invalid refresh token")`. Check expiry (`expiresAt.isBefore(Instant.now())`) — throw `BadCredentialsException("Refresh token expired")`. Then check revoked state:
       - If `revoked=true` AND `usedAt != null` AND `replacedByTokenHash != null` AND `Instant.now().isBefore(usedAt.plusSeconds(10))` — **grace window**: this is a concurrent mobile retry within 10 seconds. Throw a custom `TokenGracePeriodException("Token already rotated, retry with new token")` that extends `RuntimeException`. The controller will catch this as 409 Conflict. (See research Open Question 1 — we cannot return the replacement raw token because only the hash is stored per TOKN-02.)
       - If `revoked=true` AND outside grace window (or no `usedAt`) — **reuse detected**: call `refreshTokenRepository.deleteAllByUser(stored.user)`, then throw `BadCredentialsException("Refresh token reuse detected")`.
       - If `revoked=false` — **normal rotation**: generate new raw token + hash, save new `RefreshToken` entity, then update old token: `stored.revoked = true`, `stored.usedAt = Instant.now()`, `stored.replacedByTokenHash = newHash`, save old token. Return `Pair(stored.user, newRawToken)`.
       Mark `@Transactional`.

    3. `revoke(rawToken: String)` — Hash the incoming token. Look up by hash. If not found, return silently (idempotent). Set `stored.revoked = true`, save. Do NOT set `usedAt` (this is a logout, not a rotation). Mark `@Transactional`.

    Private helper methods:
    - `generateRawToken(): String` — `SecureRandom().nextBytes(ByteArray(32))`, `Base64.getUrlEncoder().withoutPadding().encodeToString(bytes)`
    - `hashToken(rawToken: String): String` — `MessageDigest.getInstance("SHA-256").digest(rawToken.toByteArray(Charsets.UTF_8))`, `Base64.getEncoder().encodeToString(hashBytes)`

    Use a `SecureRandom` instance field (not per-call), keep `graceWindowSeconds = 10L` as a constant.

    Constructor-inject `RefreshTokenRepository` and `@Value("\${app.auth.refresh-token.expiry-days:30}") expiryDays: Long`.

    Also create the `TokenGracePeriodException` class — can be a simple top-level class in the same file or a separate file. Simplest: define it in the same file as `class TokenGracePeriodException(message: String) : RuntimeException(message)`.

    **GlobalExceptionHandler.kt** — Add two new `@ExceptionHandler` methods BEFORE the catch-all `Exception` handler (order matters):

    ```kotlin
    @ExceptionHandler(BadCredentialsException::class)
    fun handleBadCredentials(ex: BadCredentialsException): ResponseEntity<ErrorResponse> =
        ResponseEntity.status(HttpStatus.UNAUTHORIZED).body(
            ErrorResponse(error = "Unauthorized", message = ex.message ?: "Invalid credentials", status = 401)
        )

    @ExceptionHandler(TokenGracePeriodException::class)
    fun handleGracePeriod(ex: TokenGracePeriodException): ResponseEntity<ErrorResponse> =
        ResponseEntity.status(HttpStatus.CONFLICT).body(
            ErrorResponse(error = "Conflict", message = ex.message ?: "Token already rotated", status = 409)
        )
    ```

    Add imports: `org.springframework.security.authentication.BadCredentialsException`, `org.springframework.http.HttpStatus`, `kz.innlab.template.authentication.TokenGracePeriodException`.
