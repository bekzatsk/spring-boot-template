    - `@Transactional fun authenticate(idTokenString: String): AuthResponse`
      1. `val idToken = googleIdTokenVerifier.verify(idTokenString) ?: throw BadCredentialsException("Invalid Google ID token")` — CRITICAL: `verify()` returns null on failure, not exception (Pitfall 1)

      4. `jwtTokenService.generateAccessToken(user.id!!, user.roles)` for access token
      5. `refreshTokenService.createToken(user)` for refresh token
      6. Return `AuthResponse(accessToken, refreshToken)`

    **AuthController.kt** — Create at `src/main/kotlin/kz/innlab/template/authentication/AuthController.kt`. Follow research Pattern 8:
    - `@RestController @RequestMapping("/api/v1/auth")` class
    - Constructor-inject `GoogleAuthService`, `RefreshTokenService`, `JwtTokenService`
    - Three endpoints (all under `/api/v1/auth/**` which is already `permitAll` in SecurityConfig):

    1. `@PostMapping("/google") fun googleLogin(@Valid @RequestBody request: AuthRequest): ResponseEntity<AuthResponse>` — Delegates to `googleAuthService.authenticate(request.idToken)`. Returns `ResponseEntity.ok(response)`. The existing `AuthRequest` DTO from Phase 2 has `@NotBlank` on `idToken`.

    2. `@PostMapping("/refresh") fun refresh(@Valid @RequestBody request: RefreshRequest): ResponseEntity<AuthResponse>` — Calls `refreshTokenService.rotate(request.refreshToken)` which returns `Pair<User, String>`. Then calls `jwtTokenService.generateAccessToken(user.id!!, user.roles)` for the new access token. Returns `ResponseEntity.ok(AuthResponse(accessToken, newRawToken))`.

    3. `@PostMapping("/revoke") fun revoke(@Valid @RequestBody request: RefreshRequest): ResponseEntity<Void>` — Calls `refreshTokenService.revoke(request.refreshToken)`. Returns `ResponseEntity.noContent().build()`.

    Note: `BadCredentialsException` from `RefreshTokenService.rotate()` is caught by `GlobalExceptionHandler` (Plan 01 added the handler) and returns 401 JSON. `TokenGracePeriodException` returns 409. No try/catch needed in the controller.
