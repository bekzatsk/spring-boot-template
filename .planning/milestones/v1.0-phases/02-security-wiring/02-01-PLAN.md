CRITICAL: Add `import org.springframework.security.config.annotation.web.invoke` — without this import, the Kotlin DSL `http { }` block will not compile.

SecurityFilterChain bean (`securityFilterChain(http: HttpSecurity)`):

- `cors { configurationSource = corsConfigurationSource() }` — SECU-04

Create `GlobalExceptionHandler.kt` in package `kz.innlab.template.shared.error`. Annotated `@RestControllerAdvice`.

Three handler methods:

**1. `handleValidation(ex: MethodArgumentNotValidException): ResponseEntity<ErrorResponse>`**
- Annotated `@ExceptionHandler(MethodArgumentNotValidException::class)`
- Extract field errors: `ex.bindingResult.fieldErrors.joinToString("; ") { "${it.field}: ${it.defaultMessage}" }`
- Return `ResponseEntity.badRequest().body(ErrorResponse(error = "Bad Request", message = message, status = 400))`
- This handles `@Valid @RequestBody` validation failures (SECU-08)

**2. `handleMethodValidation(ex: HandlerMethodValidationException): ResponseEntity<ErrorResponse>`**
- Annotated `@ExceptionHandler(HandlerMethodValidationException::class)`
- Extract errors: `ex.allErrors.joinToString("; ") { it.defaultMessage ?: "Invalid value" }`
- Return `ResponseEntity.badRequest().body(ErrorResponse(error = "Bad Request", message = message, status = 400))`
- Safety net for Spring Boot 3.2+ method-parameter validation (different exception type than @Valid @RequestBody)

**3. `handleGeneral(ex: Exception): ResponseEntity<ErrorResponse>`**
- Annotated `@ExceptionHandler(Exception::class)`
- Return `ResponseEntity.internalServerError().body(ErrorResponse(error = "Internal Server Error", message = "An unexpected error occurred", status = 500))`
- Do NOT expose `ex.message` in production — it may contain sensitive information

Import `org.springframework.web.bind.MethodArgumentNotValidException` and `org.springframework.web.method.annotation.HandlerMethodValidationException`.
