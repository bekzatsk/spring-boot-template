Create 3 new Kotlin files:

**1. `authentication/JwtTokenService.kt`** — Package: `kz.innlab.template.authentication`. Annotated `@Service`. Constructor-inject `jwtEncoder: JwtEncoder` (the `NimbusJwtEncoder` bean from `RsaKeyConfig`).

Method `generateAccessToken(userId: UUID, roles: Set<Role>): String`:
- `val now = Instant.now()`
- Build `JwtClaimsSet` with:
  - `.issuer("template-app")`
  - `.issuedAt(now)`
  - `.expiresAt(now.plus(15, ChronoUnit.MINUTES))` — TOKN-01: 15-minute expiry
  - `.subject(userId.toString())` — sub = user UUID string
  - `.claim("roles", roles.map { it.name })` — e.g., ["USER"], must match JwtGrantedAuthoritiesConverter config in SecurityConfig
- Return `jwtEncoder.encode(JwtEncoderParameters.from(claims)).tokenValue`
- Do NOT set JwsHeader explicitly — Nimbus defaults to RS256 when JWKSource contains an RSA key (per research Pitfall 6)

Import `kz.innlab.template.user.Role`, `org.springframework.security.oauth2.jwt.JwtClaimsSet`, `org.springframework.security.oauth2.jwt.JwtEncoder`, `org.springframework.security.oauth2.jwt.JwtEncoderParameters`, `java.time.Instant`, `java.time.temporal.ChronoUnit`, `java.util.UUID`.

**2. `authentication/dto/AuthRequest.kt`** — Package: `kz.innlab.template.authentication.dto`. A data class for social login auth requests (used by Phase 3 Google auth controller and Phase 4 Apple auth controller).

```kotlin
data class AuthRequest(
    @field:NotBlank(message = "ID token is required")
    val idToken: String = ""
)
```

Import `jakarta.validation.constraints.NotBlank`. The `= ""` default is required for Kotlin data classes with `@field:` annotations to work with Jackson deserialization (Jackson needs a no-arg constructor or default values; Kotlin data classes need explicit defaults for Jackson to populate via property setting). SECU-08: the `@NotBlank` annotation ensures a blank/missing `idToken` triggers `MethodArgumentNotValidException` when used with `@Valid @RequestBody` in a controller.

**3. `user/UserController.kt`** — Package: `kz.innlab.template.user`. Annotated `@RestController`, `@RequestMapping("/api/v1/users")`.

Method `getCurrentUser(authentication: Authentication): Map<String, Any?>` annotated `@GetMapping("/me")`:
- Cast `authentication.principal` to `org.springframework.security.oauth2.jwt.Jwt`
- Return `mapOf("sub" to jwt.subject, "roles" to jwt.getClaimAsStringList("roles"))`
- This is a STUB — Phase 3 will replace with actual UserService lookup. The stub proves SECU-02: auth passes without hitting the database, and the JWT claims are accessible.

Import `org.springframework.security.core.Authentication`, `org.springframework.security.oauth2.jwt.Jwt`.

`@Autowired` inject `MockMvc` and `JwtTokenService`.

**Test 1: `noToken_returns401WithJsonBody()`** (Success Criteria 1, 3)
- `mockMvc.perform(get("/api/v1/users/me"))`
- `.andExpect(status().isUnauthorized)`
- `.andExpect(jsonPath("$.error").value("Unauthorized"))`
- `.andExpect(jsonPath("$.status").value(401))`

**Test 2: `validToken_returns200WithJwtClaims()`** (Success Criteria 2, 3)
- Generate token: `val token = jwtTokenService.generateAccessToken(UUID.randomUUID(), setOf(Role.USER))`
- `mockMvc.perform(get("/api/v1/users/me").header("Authorization", "Bearer $token"))`
- `.andExpect(status().isOk)`
- `.andExpect(jsonPath("$.sub").exists())`
- `.andExpect(jsonPath("$.roles[0]").value("USER"))`
- This proves the full chain: JWT minted by JwtTokenService → validated by SecurityFilterChain JwtDecoder → reaches UserController → returns claims without DB hit

**Test 3: `corsPreflight_returns200WithHeaders()`** (Success Criterion 4)
- `mockMvc.perform(options("/api/v1/users/me").header("Origin", "http://localhost:3000").header("Access-Control-Request-Method", "GET").header("Access-Control-Request-Headers", "Authorization"))`
- `.andExpect(status().isOk)`
- `.andExpect(header().string("Access-Control-Allow-Origin", "http://localhost:3000"))`
