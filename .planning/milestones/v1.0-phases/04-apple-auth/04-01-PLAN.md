<objective>
Wire Apple Sign In authentication end-to-end: NimbusJwtDecoder validates Apple identity tokens via Apple's JWKS endpoint, AppleAuthService orchestrates the flow, UserService persists first-login data atomically, and AuthController exposes POST /api/v1/auth/apple.

Purpose: Enable mobile clients to authenticate with Apple ID tokens using the same token lifecycle (access + refresh) already built for Google auth.
Output: Working /auth/apple endpoint that creates users on first sign-in and authenticates returning users without requiring email in subsequent tokens.
</objective>

  <action>
    **1. Create `config/AppleAuthConfig.kt`:**
    - `@Configuration` class with `@Value("${app.auth.apple.bundle-id}")` for the audience value
    - `@Bean(name = ["appleJwtDecoder"])` method returning `NimbusJwtDecoder`
    - Use `NimbusJwtDecoder.withJwkSetUri("https://appleid.apple.com/auth/keys").jwsAlgorithm(SignatureAlgorithm.RS256).build()`
    - Set validator: `DelegatingOAuth2TokenValidator` composing `JwtTimestampValidator()`, `JwtIssuerValidator("https://appleid.apple.com")`, and a `JwtClaimValidator` for `aud` that checks `aud contains appleBundleId`
    - Imports from `org.springframework.security.oauth2.jwt.*`, `org.springframework.security.oauth2.core.*`, `org.springframework.security.oauth2.jose.jws.SignatureAlgorithm`
    - See 04-RESEARCH.md Pattern 1 for exact code

    **2. Create `authentication/dto/AppleAuthRequest.kt`:**
    - `data class AppleAuthRequest` with:
      - `@field:NotBlank(message = "ID token is required") val idToken: String = ""`
      - `val givenName: String? = null` (optional — only on first sign-in from iOS client)
      - `val familyName: String? = null` (optional — only on first sign-in from iOS client)
    - Do NOT add `@Email` annotation on any field — email comes from JWT claims, not request body. AUTH-06 is satisfied by design.

    **3. Create `authentication/AppleAuthService.kt`:**
    - `@Service` class injecting `@Qualifier("appleJwtDecoder") appleJwtDecoder: JwtDecoder`, `userService: UserService`, `jwtTokenService: JwtTokenService`, `refreshTokenService: RefreshTokenService`
    - `@Transactional fun authenticate(request: AppleAuthRequest): AuthResponse` method:
      - Decode token: `val jwt = try { appleJwtDecoder.decode(request.idToken) } catch (ex: Exception) { throw BadCredentialsException("Invalid Apple ID token: ${ex.message}", ex) }`
      - IMPORTANT: NimbusJwtDecoder throws `JwtException`, NOT `BadCredentialsException` — must wrap (Pitfall 3 from research)
      - Extract `sub` from `jwt.subject` — throw `BadCredentialsException` if null
      - Extract `email` from `jwt.getClaimAsString("email")` — may be null on subsequent logins (this is expected)
      - Extract name from request body: `request.givenName?.takeIf { it.isNotBlank() }`, same for `familyName`
      - Build `fullName` by joining non-null name parts with space, or null if both blank
      - Call `userService.findOrCreateAppleUser(providerId = sub, email = email, name = fullName)`
      - Generate tokens: `jwtTokenService.generateAccessToken(user.id!!, user.roles)` + `refreshTokenService.createToken(user)`
      - Return `AuthResponse(accessToken = accessToken, refreshToken = refreshToken)`
    - Follow the same pattern as GoogleAuthService — controllers delegate to services, no try/catch in controllers

    **4. Extend `user/UserService.kt` with `findOrCreateAppleUser`:**
    - Add import for `org.springframework.security.authentication.BadCredentialsException`
    - Add method: `@Transactional fun findOrCreateAppleUser(providerId: String, email: String?, name: String?): User`
    - First: `val existing = userRepository.findByProviderAndProviderId(AuthProvider.APPLE, providerId)` — if found, return immediately (do NOT require email/name on subsequent logins)
    - For new users: `val resolvedEmail = email ?: throw BadCredentialsException("Email not present in Apple identity token on first sign-in")` — email absent for new users means iOS client scope is misconfigured
    - Save: `userRepository.save(User(email = resolvedEmail, provider = AuthProvider.APPLE, providerId = providerId).also { it.name = name })` — name persisted atomically in same transaction (AUTH-05)
    - Do NOT set `picture` — Apple does not provide profile pictures
    - Do NOT update email or name for existing users — Apple only sends these on first login; updating would overwrite user changes
