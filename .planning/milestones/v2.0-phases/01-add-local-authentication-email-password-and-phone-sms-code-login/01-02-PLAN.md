**LocalAuthConfig.kt** (`src/main/kotlin/kz/innlab/template/config/LocalAuthConfig.kt`):
- `@Configuration` class
- `@Bean fun passwordEncoder(): PasswordEncoder = PasswordEncoderFactories.createDelegatingPasswordEncoder()` -- DelegatingPasswordEncoder defaults to BCrypt
- `@Bean fun localAuthenticationManager(localUserDetailsService: LocalUserDetailsService, passwordEncoder: PasswordEncoder): AuthenticationManager` -- creates `DaoAuthenticationProvider(localUserDetailsService)`, sets password encoder, wraps in `ProviderManager(provider)`
- Import: `org.springframework.security.authentication.AuthenticationManager`, `org.springframework.security.authentication.ProviderManager`, `org.springframework.security.authentication.dao.DaoAuthenticationProvider`, `org.springframework.security.crypto.factory.PasswordEncoderFactories`, `org.springframework.security.crypto.password.PasswordEncoder`
- IMPORTANT per Pitfall 2 in research: This `localAuthenticationManager` bean is SEPARATE from the resource server's JWT auth. Name it explicitly to avoid confusion. Inject into LocalAuthService with `@Qualifier("localAuthenticationManager")` if needed.

**LocalUserDetailsService.kt** (`src/main/kotlin/kz/innlab/template/authentication/service/LocalUserDetailsService.kt`):
- `@Service` implementing `org.springframework.security.core.userdetails.UserDetailsService`
- Constructor-inject `UserRepository`
- `override fun loadUserByUsername(email: String): UserDetails`:
  - Look up user by `userRepository.findByProviderAndProviderId(AuthProvider.LOCAL, email)` -- NOT by email alone (Pitfall 6 in research)
  - If null, throw `UsernameNotFoundException("No local account for email: $email")`
  - If `user.passwordHash` is null, throw `BadCredentialsException("No password set")`
  - Return `org.springframework.security.core.userdetails.User.withUsername(email).password(user.passwordHash!!).roles(*user.roles.map { it.name }.toTypedArray()).build()`
- This uses the scoped `(LOCAL, email)` lookup to avoid returning Google/Apple users with the same email

**LocalAuthService.kt** (`src/main/kotlin/kz/innlab/template/authentication/service/LocalAuthService.kt`):
- `@Service` with constructor injection of: `AuthenticationManager` (qualified as `localAuthenticationManager`), `UserRepository`, `PasswordEncoder`, `TokenService`, `RefreshTokenService`
- `@Transactional fun register(email: String, rawPassword: String, name: String?): AuthResponse`:
  - Check `userRepository.findByProviderAndProviderId(AuthProvider.LOCAL, email) != null` -- if exists, throw `IllegalStateException("Email already registered")` (handled as 409)
  - Create `User(email = email, provider = AuthProvider.LOCAL, providerId = email)` -- providerId = email for LOCAL email users per research
  - Set `user.name = name`, `user.passwordHash = passwordEncoder.encode(rawPassword)`
  - Save user
  - Generate access token via `tokenService.generateAccessToken(user.id!!, user.roles)`
  - Create refresh token via `refreshTokenService.createToken(user)`
  - Return `AuthResponse(accessToken, refreshToken)`
- `fun login(email: String, rawPassword: String): AuthResponse`:
  - Call `authenticationManager.authenticate(UsernamePasswordAuthenticationToken.unauthenticated(email, rawPassword))` -- throws `BadCredentialsException` on failure (caught by AuthExceptionHandler -> 401)
  - After successful auth, look up user: `userRepository.findByProviderAndProviderId(AuthProvider.LOCAL, email) ?: throw BadCredentialsException("User not found")`
  - Generate and return tokens same as register

fun localRegister(@Valid @RequestBody request: LocalRegisterRequest): ResponseEntity<AuthResponse> {
    // TODO: rate limiting
    val response = localAuthService.register(request.email, request.password, request.name)

fun localLogin(@Valid @RequestBody request: LocalLoginRequest): ResponseEntity<AuthResponse> {

}
