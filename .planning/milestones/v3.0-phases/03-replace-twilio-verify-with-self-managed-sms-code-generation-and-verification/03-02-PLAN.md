Completely rewrite `PhoneAuthIntegrationTest.kt` to test against the new self-managed SMS verification flow.

**Key architecture change:** Tests no longer mock the verification result. Instead:
1. `@MockitoBean SmsService` ŌĆö only mock the SMS delivery (ConsoleSmsService replaced in test context)
2. Use `ArgumentCaptor<String>` from Mockito to capture the actual 6-digit code passed to `smsService.sendCode(phone, code)`
3. Call `/phone/request` endpoint (which triggers real SmsVerificationService.sendCode -> real BCrypt hash stored in H2)
4. Extract the captured code from ArgumentCaptor
5. Call `/phone/verify` with the captured code (which triggers real SmsVerificationService.verifyCode -> real BCrypt matches against H2)

**Imports (use these exact imports ŌĆö Spring Boot 4.x / Spring 7.x packages):**
```kotlin
import org.springframework.boot.test.context.SpringBootTest
import org.springframework.boot.webmvc.test.autoconfigure.AutoConfigureMockMvc
import org.springframework.test.context.bean.override.mockito.MockitoBean
import org.mockito.ArgumentCaptor
import org.mockito.Mockito.anyString
import org.mockito.Mockito.doNothing
import org.mockito.Mockito.verify as mockitoVerify
```

**Injected dependencies:**
- `@MockitoBean private lateinit var smsService: SmsService` ŌĆö replaces `twilioVerifyClient`
- `@Autowired private lateinit var mockMvc: MockMvc`

- `@Autowired private lateinit var refreshTokenRepository: RefreshTokenRepository`
- `@Autowired private lateinit var smsVerificationRepository: SmsVerificationRepository` ŌĆö needed for cleanup in @BeforeEach

**@BeforeEach cleanup:**

**Test cases to implement:**

1. **`request OTP success returns 204`**
   - `doNothing().when(smsService).sendCode(anyString(), anyString())`
   - POST `/api/v1/auth/phone/request` with `{"phone": "$testPhone"}`
   - Assert 204 No Content

2. **`verify OTP success for new user creates user and returns tokens`**
   - Setup: `doNothing().when(smsService).sendCode(anyString(), anyString())`
   - Step 1: POST `/api/v1/auth/phone/request` with `{"phone": "$testPhone"}`
   - Step 2: Capture code with `ArgumentCaptor.forClass(String::class.java)`, call `mockitoVerify(smsService).sendCode(anyString(), codeCaptor.capture())`, extract `val code = codeCaptor.value`
   - Step 3: POST `/api/v1/auth/phone/verify` with `{"phone": "$testPhone", "code": "$code"}`
   - Assert 200 with accessToken and refreshToken in response
   - Assert user created in DB with LOCAL provider and correct phone

3. **`verify OTP success for returning user finds existing user and returns tokens`**
   - Pre-create user: `userRepository.save(User(email = "").also { it.providers.add(AuthProvider.LOCAL); it.phone = testPhone })`
   - Same request-then-verify flow as test 2 using ArgumentCaptor
   - Assert exactly 1 user with that phone exists (no duplicate)
   - Assert same user ID as pre-created user

4. **`verify OTP failure with wrong code returns 401`**
   - Step 1: POST `/api/v1/auth/phone/request` (to create a real verification record)
   - Step 2: POST `/api/v1/auth/phone/verify` with wrong code `"000000"` (or any non-matching code)
   - Assert 401 Unauthorized with error body

5. **`request OTP with empty phone returns 400`**
   - POST `/api/v1/auth/phone/request` with `{"phone": ""}`
   - Assert 400 Bad Request (validation error)

6. **`verify OTP with invalid phone format returns 400`**
   - POST `/api/v1/auth/phone/verify` with `{"phone": "not-a-number", "code": "123456"}`
   - Assert 400 Bad Request

7. **`request OTP rate limited returns 409`** (tests the 60-second rate limit)
   - Setup: `doNothing().when(smsService).sendCode(anyString(), anyString())`
   - Step 1: POST `/api/v1/auth/phone/request` with `{"phone": "$testPhone"}` ŌĆö assert 204
   - Step 2: POST `/api/v1/auth/phone/request` with `{"phone": "$testPhone"}` immediately again ŌĆö assert 409 Conflict (rate limit hit, IllegalStateException caught by AuthExceptionHandler)
