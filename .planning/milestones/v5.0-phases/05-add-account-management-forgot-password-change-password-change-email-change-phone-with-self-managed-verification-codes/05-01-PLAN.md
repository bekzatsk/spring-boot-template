<objective>
Create the shared verification code infrastructure for all account management flows.

Purpose: All four account management operations (forgot password, change password, change email, change phone) share the same verification code lifecycle: generate -> hash -> store -> verify -> expire -> cleanup. This plan builds that reusable infrastructure, mirroring the existing SmsVerificationService/SmsVerification pattern but generalized with a `purpose` discriminator column.

Output: VerificationCode entity, VerificationCodeRepository, V2 Flyway migration, EmailService interface, ConsoleEmailService, VerificationCodeService, updated SmsSchedulerConfig with email bean and verification_codes cleanup.
</objective>

```kotlin
interface VerificationCodeRepository : JpaRepository<VerificationCode, UUID> {
    fun existsByIdentifierAndPurposeAndCreatedAtAfter(
        identifier: String,
        purpose: VerificationPurpose,
        since: Instant
    ): Boolean

    @Modifying
    @Transactional
    @Query("DELETE FROM VerificationCode vc WHERE vc.expiresAt < :cutoff OR vc.used = true")
    fun deleteExpiredOrUsed(@Param("cutoff") cutoff: Instant)

    @Modifying
    @Transactional
    @Query("DELETE FROM VerificationCode vc WHERE vc.identifier = :identifier AND vc.purpose = :purpose")
    fun deleteAllByIdentifierAndPurpose(
        @Param("identifier") identifier: String,
        @Param("purpose") purpose: VerificationPurpose
    )
}
```

**4. V2__add_verification_codes.sql** — in `resources/db/migration/`:
```sql
CREATE TABLE verification_codes (
    id UUID PRIMARY KEY,
    user_id UUID REFERENCES users(id),
    identifier VARCHAR(255) NOT NULL,
    purpose VARCHAR(50) NOT NULL,
    code_hash VARCHAR(255) NOT NULL,
    new_value VARCHAR(255),
    expires_at TIMESTAMPTZ NOT NULL,
    used BOOLEAN NOT NULL DEFAULT FALSE,
    attempts INTEGER NOT NULL DEFAULT 0,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_verification_codes_identifier_purpose
    ON verification_codes(identifier, purpose);
```

IMPORTANT: Use `VARCHAR(50)` for `purpose` column, NOT PostgreSQL `CREATE TYPE ... AS ENUM` — H2 does not support PostgreSQL native enum types. This is consistent with the existing schema pattern (e.g., `provider VARCHAR(50)` in user_providers).

**3. VerificationCodeService.kt** — `@Service` in `authentication/service/`. Mirror `SmsVerificationService` structure:
- Constructor injection: `VerificationCodeRepository`, `PasswordEncoder`, `@Value("\${app.auth.verification.dev-code:}") devCode: String = ""`
- Companion object constants: `CODE_BOUND = 1_000_000`, `EXPIRY_MINUTES = 15L` (longer than SMS 5 min — user needs time to check email), `RATE_LIMIT_SECONDS = 60L`, `MAX_ATTEMPTS = 3`, `random = SecureRandom()`
- `@Transactional fun createCode(identifier: String, purpose: VerificationPurpose, newValue: String? = null, userId: UUID? = null): Pair<UUID, String>`:
  - Rate limit check: `if (repository.existsByIdentifierAndPurposeAndCreatedAtAfter(identifier, purpose, Instant.now().minusSeconds(RATE_LIMIT_SECONDS)))` throw `IllegalStateException("Please wait before requesting a new code")`
  - Generate code: `if (devCode.isNotBlank()) devCode else String.format("%06d", random.nextInt(CODE_BOUND))`
  - Hash: `passwordEncoder.encode(code)!!` (Java method returns String? in Kotlin — !! is correct per Phase 03-01 decision)
  - Delete existing codes for same identifier+purpose: `repository.deleteAllByIdentifierAndPurpose(identifier, purpose)`
  - Save new `VerificationCode(identifier, purpose, hash, Instant.now().plusSeconds(EXPIRY_MINUTES * 60), newValue, userId)`
  - Return `Pair(saved.id, code)` — caller sends the raw code via EmailService or SmsService
- `@Transactional fun verifyCode(verificationId: UUID, identifier: String, purpose: VerificationPurpose, code: String): VerificationCode`:
  - Find by ID: `repository.findById(verificationId).orElseThrow { BadCredentialsException("Invalid verification code") }`
  - Validate: identifier matches, purpose matches, not used, not expired, attempts < MAX_ATTEMPTS
  - Increment attempts before checking code (prevents brute force): `record.attempts++; repository.save(record)`
  - Check `passwordEncoder.matches(code, record.codeHash)` — if false, throw `BadCredentialsException("Invalid verification code")`
  - Mark used: `record.used = true; repository.save(record)`
  - Return the verified `VerificationCode` record (caller reads `newValue`, `userId`, etc.)
  - On any validation failure (wrong identifier, expired, used, max attempts), throw `BadCredentialsException("Invalid verification code")` — consistent 401 for all invalid codes
