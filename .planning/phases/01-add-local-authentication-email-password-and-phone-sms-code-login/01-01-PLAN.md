**V1__initial_schema.sql** -- Create `src/main/resources/db/migration/V1__initial_schema.sql` capturing the existing Hibernate-generated schema for PostgreSQL:

```sql
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    email VARCHAR(255) NOT NULL,
    provider VARCHAR(50) NOT NULL,
    provider_id VARCHAR(255) NOT NULL,
    name VARCHAR(255),
    picture VARCHAR(512),
    password_hash VARCHAR(255),
    phone VARCHAR(30) UNIQUE,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    CONSTRAINT uq_provider_provider_id UNIQUE (provider, provider_id)
);

CREATE TABLE user_roles (
    user_id UUID NOT NULL REFERENCES users(id),
    role VARCHAR(50) NOT NULL
);

CREATE TABLE refresh_tokens (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID NOT NULL REFERENCES users(id),
    token_hash VARCHAR(512) NOT NULL UNIQUE,
    expires_at TIMESTAMPTZ NOT NULL,
    revoked BOOLEAN NOT NULL DEFAULT FALSE,
    used_at TIMESTAMPTZ,
    replaced_by_token_hash VARCHAR(512),
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

Note: V1 includes `password_hash` and `phone` columns already since this is a fresh template. We combine the initial schema into one migration rather than requiring V2 for a greenfield project. However, also create V2 as a no-op or skip V2 entirely.

Actually -- since this is a template project with no production data and `ddl-auto: create-drop` was used in dev, V1 should capture the COMPLETE target schema (including local auth columns). There is no need for a V2. Create only V1__initial_schema.sql with all columns.

**application.yaml** -- Update the dev profile to use Flyway and switch `ddl-auto` from `create-drop` to `validate`:

In the dev profile section (after `---`), add Flyway config and change ddl-auto:
```yaml
spring:
  config:
    activate:
      on-profile: dev
  datasource:
    url: jdbc:postgresql://localhost:5432/${DB_NAME:template}
    username: ${DB_USERNAME:postgres}
    password: ${DB_PASSWORD:postgres}
  flyway:
    enabled: true
    locations: classpath:db/migration
  jpa:
    hibernate:
      ddl-auto: validate
    show-sql: true
    properties:
      hibernate:
        format_sql: true
```

In the prod profile section, add Flyway config (it should already have `ddl-auto: validate`):
```yaml
spring:
  config:
    activate:
      on-profile: prod
  datasource:
    url: ${DATABASE_URL}
    username: ${DB_USERNAME}
    password: ${DB_PASSWORD}
  flyway:
    enabled: true
    baseline-on-migrate: false
    validate-on-migrate: true
    out-of-order: false
  jpa:
    hibernate:
      ddl-auto: validate
```
