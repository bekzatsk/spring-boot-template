       class MailController(
           private val mailService: MailService,

       ) {

       NOTE: The mailService used here needs to be the SmtpMailService (or ConsoleMailService) which has a `sendEmail(userId, ...)` method. Since MailService interface only has `send(to, subject, ...)`, we need to either:
       - Inject SmtpMailService directly and use it for the send-with-tracking flow, OR
       - Add a `sendEmail(userId, ...)` method to the MailService interface that returns UUID

       **Best approach**: Add `sendEmail` to MailService interface:
       ```kotlin
       interface MailService {
           fun send(to: String, subject: String, textBody: String? = null, htmlBody: String? = null, attachments: List<EmailAttachment> = emptyList())
           fun sendEmail(userId: UUID, to: String, subject: String, textBody: String? = null, htmlBody: String? = null, attachments: List<EmailAttachment> = emptyList()): UUID
       }
       ```
       ConsoleMailService.sendEmail returns a generated UUID. SmtpMailService.sendEmail saves history and dispatches.

       Endpoints:
       - `POST /api/v1/mail/send` — Accept multipart/form-data:
         - `@RequestPart("email") request: SendEmailRequest` (JSON part)
         - `@RequestPart("files", required = false) files: List<MultipartFile>?`
         - Extract userId from JWT, convert MultipartFile to EmailAttachment, call `mailService.sendEmail(userId, ...)`
         - Return `ResponseEntity.accepted().body(mapOf("mailId" to mailId))`
         - Also accept plain JSON (no files): `@PostMapping("/send", consumes = [MediaType.APPLICATION_JSON_VALUE])` as a separate endpoint or use conditional logic. Simplest: make the files part optional in multipart. OR have a single endpoint that accepts both JSON-only and multipart. Use `@PostMapping("/send")` with `@RequestBody` for JSON-only case and a separate `@PostMapping("/send/with-attachments")` for multipart. Actually, simplest: single endpoint accepting multipart with optional files part.

       - `GET /api/v1/mail/inbox` — Query params: `offset` (default 0), `size` (default 20), `unreadOnly` (default false)
         - Calls `imapService.listInbox(offset, size, unreadOnly)`
         - Returns `ResponseEntity.ok(page)` with InboxMessageResponse list + total/offset/size wrapper

       - `GET /api/v1/mail/inbox/{messageNumber}` — Fetch single email
         - Calls `imapService.getMessage(messageNumber)`
         - Returns `ResponseEntity.ok(EmailMessageResponse.from(msg))`

       - `PUT /api/v1/mail/inbox/{messageNumber}/read` — Mark read
         - Calls `imapService.markRead(messageNumber)`
         - Returns `ResponseEntity.ok().build()`

       - `DELETE /api/v1/mail/inbox/{messageNumber}/read` — Mark unread
         - Calls `imapService.markUnread(messageNumber)`
         - Returns `ResponseEntity.ok().build()`

       - `GET /api/v1/mail/history` — Mail send history for current user
         - Query params: `cursor` (UUID, optional), `size` (default 20, capped 1-100)
         - Calls `mailHistoryRepository.findByUserIdLatest/BeforeCursor`
         - Returns list of MailHistoryResponse

    8. **NotificationController.kt** — ADD preference endpoints to existing controller:
       - Add `NotificationPreferenceService` to constructor injection
       - `GET /api/v1/notifications/preferences`:
         ```kotlin
         @GetMapping("/preferences")
         fun getPreferences(@AuthenticationPrincipal jwt: Jwt): ResponseEntity<NotificationPreferenceResponse> {
             val userId = UUID.fromString(jwt.subject)
             val prefs = notificationPreferenceService.getPreferences(userId)
             return ResponseEntity.ok(NotificationPreferenceResponse(
                 push = prefs[NotificationChannel.PUSH] ?: true,
                 email = prefs[NotificationChannel.EMAIL] ?: true
             ))
         }
         ```
       - `PUT /api/v1/notifications/preferences`:
         ```kotlin
         @PutMapping("/preferences")
         fun updatePreferences(
             @Valid @RequestBody request: NotificationPreferenceRequest,
             @AuthenticationPrincipal jwt: Jwt
         ): ResponseEntity<NotificationPreferenceResponse> {
             val userId = UUID.fromString(jwt.subject)
             val updates = mutableMapOf<NotificationChannel, Boolean>()
             request.push?.let { updates[NotificationChannel.PUSH] = it }
             request.email?.let { updates[NotificationChannel.EMAIL] = it }
             notificationPreferenceService.updatePreferences(userId, updates)
             val prefs = notificationPreferenceService.getPreferences(userId)
             return ResponseEntity.ok(NotificationPreferenceResponse(
                 push = prefs[NotificationChannel.PUSH] ?: true,
                 email = prefs[NotificationChannel.EMAIL] ?: true
             ))
         }
         ```
