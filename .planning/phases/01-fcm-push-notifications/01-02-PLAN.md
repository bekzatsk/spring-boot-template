1. **PushService.kt** — Interface in `kz.innlab.template.notification.service`:
```kotlin
interface PushService {
    fun sendToToken(token: String, title: String, body: String, data: Map<String, String>): String?
    fun sendMulticast(tokens: List<String>, title: String, body: String, data: Map<String, String>): List<String>
    fun sendToTopic(topic: String, title: String, body: String, data: Map<String, String>): String?
    fun subscribeToTopic(tokens: List<String>, topic: String)
    fun unsubscribeFromTopic(tokens: List<String>, topic: String)
}
```
- `sendToToken` returns message ID on success, null on console fallback
- `sendMulticast` returns list of stale token FCM tokens that should be deleted (empty if none or console)
- `sendToTopic` returns message ID on success

2. **FirebasePushService.kt** — `@Service` with `@ConditionalOnProperty(name = ["app.firebase.enabled"], havingValue = "true")`:
- Package: `kz.innlab.template.notification.service`
- Implements `PushService`
- Logger via `LoggerFactory.getLogger(FirebasePushService::class.java)`

- `sendToToken()`: Build `Message` with `Notification.builder().setTitle(title).setBody(body).build()`, `.putAllData(data)`, `.setToken(token)`. Call `FirebaseMessaging.getInstance().send(message)`. Catch `FirebaseMessagingException`: if `messagingErrorCode` is `MessagingErrorCode.UNREGISTERED` or `MessagingErrorCode.INVALID_ARGUMENT`, log warning and re-throw. Otherwise log error and re-throw. Return message ID string.

- `sendMulticast()`: Build `MulticastMessage` with `.setNotification(...)`, `.putAllData(data)`, `.addAllTokens(tokens)`. Call `FirebaseMessaging.getInstance().sendEachForMulticast(multicast)`. Extract stale tokens: iterate `batchResponse.responses` with index, for each unsuccessful response where `exception?.messagingErrorCode` is `UNREGISTERED` or `INVALID_ARGUMENT`, collect `tokens[index]`. Return the stale tokens list.

- `sendToTopic()`: Build `Message` with `.setTopic(topic)`, `.setNotification(...)`, `.putAllData(data)`. Call `FirebaseMessaging.getInstance().send(message)`. Return message ID.

- `subscribeToTopic()`: Call `FirebaseMessaging.getInstance().subscribeToTopic(tokens, topic)`. Log success/failure counts from `TopicManagementResponse`.

- `unsubscribeFromTopic()`: Call `FirebaseMessaging.getInstance().unsubscribeFromTopic(tokens, topic)`. Log success/failure counts.

1. **DeviceTokenService.kt** — `@Service` in `kz.innlab.template.notification.service`:
- Constructor injection: `DeviceTokenRepository`, `@Value("${app.notification.token.max-per-user:5}") maxTokensPerUser: Int`
- `register(userId: UUID, platform: Platform, fcmToken: String, deviceId: String): DeviceToken`:
  - Check if token with same userId+deviceId exists (findByUserIdAndDeviceId). If exists, update fcmToken and updatedAt (upsert pattern — handles app reinstall/token refresh without duplicate rows). Return the updated entity.
  - If not exists, check `countByUserId(userId) >= maxTokensPerUser`. If so, throw `IllegalStateException("Maximum device tokens ($maxTokensPerUser) reached for user")`.
  - Create new `DeviceToken(userId, platform, fcmToken, deviceId)`. Save and return.
- `deleteByDeviceId(userId: UUID, deviceId: String)`: Find by userId+deviceId, if exists call `deviceTokenRepository.delete(it)`. If not found, do nothing (idempotent).
- `deleteAllByUser(userId: UUID)`: Call `deviceTokenRepository.deleteByUserId(userId)`.
- `listByUser(userId: UUID): List<DeviceToken>`: Return `deviceTokenRepository.findByUserId(userId)`.
- `cleanupStaleTokens(staleTokens: List<String>)`: If list is not empty, call `deviceTokenRepository.deleteAllByFcmTokenIn(staleTokens)`. Log the count at INFO.

2. **NotificationDispatcher.kt** — `@Service` in `kz.innlab.template.notification.service`:
- **This is a SEPARATE bean to avoid @Async self-invocation proxy bypass** (Pitfall 4 in research).
- Constructor injection: `PushService`, `NotificationHistoryRepository`, `DeviceTokenService`
- `@Async @Transactional fun dispatchToToken(historyId: UUID, token: String, title: String, body: String, data: Map<String, String>)`:
  - Load history: `notificationHistoryRepository.findById(historyId).orElseThrow()`
  - try: `pushService.sendToToken(token, title, body, data)` -> set `history.status = NotificationStatus.SENT`
  - catch (e: Exception): Check if it's a stale token error (FirebaseMessagingException with UNREGISTERED/INVALID_ARGUMENT) — if so, call `deviceTokenService.cleanupStaleTokens(listOf(token))`. Set `history.status = NotificationStatus.FAILED`.
  - finally: `notificationHistoryRepository.save(history)`

- `@Async @Transactional fun dispatchMulticast(historyId: UUID, tokens: List<String>, title: String, body: String, data: Map<String, String>)`:
  - Load history by ID
  - try: `val staleTokens = pushService.sendMulticast(tokens, title, body, data)` -> cleanup stale tokens -> set status SENT
  - catch: set status FAILED
  - finally: save history

- `@Async @Transactional fun dispatchToTopic(historyId: UUID, topic: String, title: String, body: String, data: Map<String, String>)`:
  - Load history by ID
  - try: `pushService.sendToTopic(topic, title, body, data)` -> status SENT
  - catch: status FAILED
  - finally: save history

3. **NotificationService.kt** — `@Service` in `kz.innlab.template.notification.service`:
- Constructor injection: `NotificationHistoryRepository`, `NotificationDispatcher`

- `sendToToken(userId: UUID, token: String, title: String, body: String, data: Map<String, String>): UUID`:
  - Create `NotificationHistory(userId, NotificationType.SINGLE, token, title, body)` with `data` set as JSON string (use simple `data.toString()` or serialize if needed — research says store as TEXT)
  - Save with status PENDING (default)
  - Call `notificationDispatcher.dispatchToToken(history.id, token, title, body, data)`
  - Return `history.id` (for 202 Accepted response)

- `sendMulticast(userId: UUID, tokens: List<String>, title: String, body: String, data: Map<String, String>): UUID`:
  - Validate `tokens.size <= 500` — throw `IllegalArgumentException("Maximum 500 tokens per multicast")`
  - Create NotificationHistory with type MULTICAST, recipient = tokens.joinToString(",")
  - Save PENDING, dispatch async, return ID

- `sendToTopic(userId: UUID, topic: String, title: String, body: String, data: Map<String, String>): UUID`:
  - Create NotificationHistory with type TOPIC, recipient = topic
  - Save PENDING, dispatch async, return ID

- `getHistory(userId: UUID, cursor: UUID?, size: Int): List<NotificationHistory>`:
  - `val pageable = PageRequest.of(0, size)`
  - If cursor is null: `notificationHistoryRepository.findByUserIdLatest(userId, pageable)`
  - If cursor is present: `notificationHistoryRepository.findByUserIdBeforeCursor(userId, cursor, pageable)`

4. **TopicService.kt** — `@Service` in `kz.innlab.template.notification.service`:
- Constructor injection: `NotificationTopicRepository`, `PushService`

- `createTopic(name: String): NotificationTopic`:
  - Check `existsByName(name)` — throw `IllegalStateException("Topic '$name' already exists")` if true
  - Create and save `NotificationTopic(name)`

- `deleteTopic(name: String)`:
  - Find by name — throw `IllegalArgumentException("Topic '$name' not found")` if null
  - Delete the entity

- `validateTopicExists(name: String)`:

  - Called by controllers before subscribe/send-to-topic operations

- `subscribe(token: String, topicName: String)`:
  - `validateTopicExists(topicName)`
  - `pushService.subscribeToTopic(listOf(token), topicName)`

- `unsubscribe(token: String, topicName: String)`:
  - `validateTopicExists(topicName)`
  - `pushService.unsubscribeFromTopic(listOf(token), topicName)`
