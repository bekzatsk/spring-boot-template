**1. Create `SmsVerification.kt`** (mirrors RefreshToken entity pattern):
- JPA entity mapped to `sms_verifications` table
- Constructor params: `phone: String`, `codeHash: String`, `expiresAt: Instant`
- `@Id @GeneratedValue(strategy = GenerationType.UUID) val id: UUID? = null`
- `var used: Boolean = false` — marks consumed codes
- `var attempts: Int = 0` — tracks verification attempts (max 3)
- `@CreationTimestamp @Column(name = "created_at", nullable = false, updatable = false) val createdAt: Instant? = null`
- Manual `equals/hashCode` on `id` field only (same as RefreshToken/User pattern)
- `phone` is NOT unique — multiple rows can exist per phone until cleanup
- No FK to `users` table — phone user may not exist yet at OTP request time

**2. Create `SmsVerificationRepository.kt`**:
- Extends `JpaRepository<SmsVerification, UUID>`
- `findActiveByPhone(phone: String, now: Instant): SmsVerification?` — JPQL: `SELECT sv FROM SmsVerification sv WHERE sv.phone = :phone AND sv.used = false AND sv.expiresAt > :now AND sv.attempts < 3` (max 3 attempts)
- `deleteAllByPhone(phone: String)` — `@Modifying @Transactional @Query("DELETE FROM SmsVerification sv WHERE sv.phone = :phone")` — JPQL bulk delete (no N+1), same pattern as RefreshTokenRepository.deleteAllByUser
- `deleteExpiredOrUsed(cutoff: Instant)` — `@Modifying @Transactional @Query("DELETE FROM SmsVerification sv WHERE sv.expiresAt < :cutoff OR sv.used = true")` — for scheduled cleanup
- `existsByPhoneAndCreatedAtAfter(phone: String, since: Instant): Boolean` — for rate limiting (1 request per phone per 60 seconds)

**3. Create `SmsService.kt`** (interface):
- `fun sendCode(phone: String, code: String)` — single method interface
- No annotations on the interface itself

**4. Create `ConsoleSmsService.kt`**:
- `@Component` annotation (NOT `@ConditionalOnMissingBean` — that annotation goes on a `@Bean` factory method, not `@Component` classes; instead use `@ConditionalOnMissingBean(SmsService::class)` on a `@Configuration` + `@Bean` factory method pattern, OR simpler: just use `@Component` and document that a real SMS provider implementation should replace this by defining a `SmsService` bean which will take precedence with `@Primary`)
- Actually, the cleanest approach: Use `@Component @ConditionalOnMissingBean(SmsService::class)` — but `@ConditionalOnMissingBean` works on `@Configuration` `@Bean` methods. For `@Component` classes, we need a different approach. Use: create a `@Configuration` class with a `@Bean @ConditionalOnMissingBean(SmsService::class)` method that returns `ConsoleSmsService()`. OR simpler: just annotate `ConsoleSmsService` with `@Component` — when a real SMS provider is added later, it will be annotated `@Primary` or `ConsoleSmsService` will be profile-scoped. For this template, `@Component` is sufficient. Log with SLF4J: `logger.info("[SMS] Sending code {} to {}", code, phone)`

**5. Create `SmsVerificationService.kt`**:
- `@Service` with constructor injection of `SmsVerificationRepository`, `SmsService`, `PasswordEncoder`
- Companion object: `private const val CODE_BOUND = 1_000_000` (6-digit), `private const val EXPIRY_MINUTES = 5L`, `private const val RATE_LIMIT_SECONDS = 60L`, `private const val MAX_ATTEMPTS = 3`, `private val random = SecureRandom()`
- `@Transactional fun sendCode(phoneE164: String)`:
  1. Check rate limit: `if (smsVerificationRepository.existsByPhoneAndCreatedAtAfter(phoneE164, Instant.now().minusSeconds(RATE_LIMIT_SECONDS))) throw IllegalStateException("Please wait before requesting a new code")`
  2. Generate: `val code = String.format("%06d", random.nextInt(CODE_BOUND))`
  3. Hash: `val hash = passwordEncoder.encode(code)`
  4. Delete existing: `smsVerificationRepository.deleteAllByPhone(phoneE164)`
  5. Save: `smsVerificationRepository.save(SmsVerification(phone = phoneE164, codeHash = hash, expiresAt = Instant.now().plusSeconds(EXPIRY_MINUTES * 60)))`
  6. Send: `smsService.sendCode(phoneE164, code)`
- `@Transactional fun verifyCode(phoneE164: String, code: String): Boolean`:
  1. Find active: `val record = smsVerificationRepository.findActiveByPhone(phoneE164, Instant.now()) ?: return false`
  2. Increment attempts: `record.attempts++` then `smsVerificationRepository.save(record)`
  3. Match: `if (!passwordEncoder.matches(code, record.codeHash)) return false`
  4. Mark used: `record.used = true` then `smsVerificationRepository.save(record)`
  5. Return `true`

**6. Create `SmsSchedulerConfig.kt`**:
- `@Configuration @EnableScheduling`
- Constructor inject `SmsVerificationRepository`
- `@Scheduled(fixedRate = 600_000)` (10 minutes in ms) `@Transactional fun cleanupExpiredCodes()` — calls `smsVerificationRepository.deleteExpiredOrUsed(Instant.now())`
- Add companion object with SLF4J logger, log at debug level when cleanup runs

**7. Create `V3__add_sms_verifications.sql`**:
```sql
CREATE TABLE sms_verifications (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    phone VARCHAR(30) NOT NULL,
    code_hash VARCHAR(255) NOT NULL,
    expires_at TIMESTAMPTZ NOT NULL,
    used BOOLEAN NOT NULL DEFAULT FALSE,
    attempts INTEGER NOT NULL DEFAULT 0,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_sms_verifications_phone ON sms_verifications(phone);
```
